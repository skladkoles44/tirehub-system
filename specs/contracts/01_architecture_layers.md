# Архитектурные слои данных

## Raw Intake Layer
- Хранит факт получения и оригинальный payload.
- Не содержит бизнес-нормализации.
- Является исходным слоем для последующей обработки.
- Считается неизменяемым слоем факта получения.

## Normalized Supplier Offers Current
- Хранит текущее каноническое состояние предложений поставщика.
- Является результатом нормализации входящих данных.
- Отражает supplier reality, а не витринную модель.

## Marketplace Current
- Хранит текущее витринное состояние после matching и publish policy.
- Содержит только публикабельные офферы.
- Не является прямым продолжением raw intake без промежуточных этапов.

## Правила
- Переходы между слоями однонаправленные.
- Ingestion не пишет напрямую в marketplace layer.
- Supplier layer и marketplace layer являются разными bounded contexts.

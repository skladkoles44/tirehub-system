# Idempotency Contract

## Назначение
Данный файл фиксирует правила безопасной повторной обработки.
Одинаковый входной объект не должен создавать дубли или приводить к многократному применению одного и того же результата.

## Принцип
- Одинаковый входной объект не порождает дубли.
- Повторная обработка того же payload приводит к тому же результату.
- Повторная обработка должна быть безопасной для current update.

## Формула idempotency_key

idempotency_key = supplier_id + source_type + external_id_or_sha256 + parser_version + normalizer_version

## Важно
- Изменение `parser_version` или `normalizer_version` делает результат новым вычислением, а не дублем.
- Один и тот же payload при тех же pinned версиях должен давать тот же результат.

## Применение
| Этап | Idempotency key |
|------|-----------------|
| Intake | sha256 артефакта |
| Atomic extraction | idempotency_key + row_index |
| Normalization | idempotency_key + row_id |
| Current update | offer_key |

## Гарантии
- Повторная обработка одного и того же intake object не создаёт дубликаты atomic rows.
- Повторная нормализация не создаёт дубликаты normalized rows.
- Повторное обновление current state не создаёт дубликаты текущих офферов.
- Идемпотентность update определяется логикой `offer_key`.

## Ограничение
Если меняется код обработки и pinned версии изменяются, это считается новым вычислением, а не повтором старого результата.

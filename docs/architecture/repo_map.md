REPO MAP. Документ объясняет назначение основных каталогов репозитория Tirehub ETL. ROOT FILES. README.md содержит краткое описание системы ingestion stage. ARCHITECTURE_v1_LOCK.md содержит текущую зафиксированную архитектурную точку. Makefile содержит служебные команды проекта. CONFIG. Каталог config содержит системные конфигурации и supplier registry. DOCS. Каталог docs содержит contracts, spec, etl notes, serving docs, status docs и архивные документы. ETL_OPS. Каталог etl_ops содержит deployment and provision configuration. GOLDEN. Каталог golden содержит golden NDJSON datasets для тестов. MAPPINGS. Каталог mappings/suppliers содержит supplier mapping configuration. RULESETS. Каталог rulesets содержит data quality and normalization policies. SCHEMAS. Каталог schemas содержит canonical и rejected JSON schemas. SCRIPTS. Каталог scripts содержит executable logic системы. Подкаталоги: connectors, ingestion, curated, serving, ops, smoke, dev. SQL. Каталог sql содержит schema files, migrations and serving schema. RUNTIME-LIKE DIRECTORIES IN REPO. В репозитории сейчас присутствуют curated_v1, offers_v1, inputs, out, var, diag. Эти каталоги содержат runtime-like structure или placeholder artifacts и должны быть пересмотрены на предмет вынесения из repo. ARTIFACTS. Файлы bundle и diagnostic dump в корне репозитория должны считаться кандидатами на вынос в artifact storage. ENGINEERING RULE. Repo должен быть источником кода, конфигураций, contracts, schemas и docs. Runtime state должен жить вне repo в VAR layer.

## 1. Что произошло

Полный аудит и харденинг ETL-скрипта 4tochki. Устранены 8 уязвимостей v3.5.4.

## 2. Найденные и исправленные уязвимости

| # | Проблема | Решение |
|---|----------|---------|
| 1 | Race condition параллельных запусков | Advisory Lock (pg_try_advisory_lock) |
| 2 | code != ALL(%s) с пустым массивом | Temp Table + NOT EXISTS |
| 3 | UNNEST на 500k+ offers | Temp Table деактивация с батчингом |
| 4 | Единый Zeep Client на 6 потоков | Thread-local клиенты |
| 5 | SIGTERM не прерывает retry | ShutdownRequested + stop_if_shutdown |
| 6 | Список всех ключей offers в памяти | Temp Table вместо Python-списка |
| 7 | Двойной traceback в логах | logger.error вместо logger.exception |
| 8 | Падение при отсутствии _etl_state | CREATE IF NOT EXISTS |
